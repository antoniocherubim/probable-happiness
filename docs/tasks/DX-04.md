# DX-04 — Continuação segura após limite de iterações

Status: implementada e validada localmente.

## Resultado

Runs bloqueados exclusivamente por `max_review_iterations` podem receber um
orçamento adicional explícito sem alterar `run.json`:

```bash
./agent-loop resume \
  --run-dir /state/projects/<repo>/runs/<run> \
  --additional-iterations 3
```

O executor recomeça em `N+1` e recebe integralmente `review-N.json`. Validações,
reviewer, gate humano e delivery DX-03 permanecem obrigatórios.

## Segurança e auditoria

- incremento permitido: 1–20; limite efetivo acumulado: 50;
- autorização serializada pelo `.resume.lock`;
- `iteration-budget.json` atômico preserva limite original, cadeia de extensões,
  origem, timestamp, iteração e identificador idempotente;
- hashes vinculam cada autorização ao feedback e snapshot revisados;
- repetição após interrupção não soma orçamento;
- drift, corrupção, outros motivos de `BLOCKED` e estados de
  aprovação/delivery são recusados;
- o motivo novo é `max_review_iterations`; `max_iterations` é aceito somente
  como compatibilidade para runs antigos elegíveis;
- nenhum botão Telegram foi adicionado; essa segunda superfície fica como
  follow-up separado.
